# Task 002 — Python Package Setup

## Goal

Configure `pyproject.toml` as the single source of truth for project metadata, dependencies, and tool settings. Ensure `uv sync` produces a reproducible environment.

## Files to Create or Modify

- `pyproject.toml` — full configuration
- `uv.lock` — generated by `uv sync` (do not edit by hand)

## Exact Expected Behavior

`pyproject.toml` must define:

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "latex-article-generator"
version = "1.00"
requires-python = ">=3.10"
dependencies = [
    "crewai>=0.80.0",
    "python-dotenv>=1.0.0",
]

[project.optional-dependencies]
dev = ["pytest>=8.0.0", "pytest-cov>=5.0.0", "ruff>=0.4.0"]

[tool.hatch.build.targets.wheel]
packages = ["src/latex_article_generator"]

[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "--cov=src --cov-report=term-missing"

[tool.coverage.run]
source = ["src"]
omit = ["src/main.py", "*/tests/*"]

[tool.coverage.report]
fail_under = 85

[tool.ruff]
line-length = 100
target-version = "py310"

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "UP", "B", "C4", "SIM"]
ignore = ["E501"]
```

After `uv sync --all-extras`:
- `crewai`, `python-dotenv`, `pytest`, `pytest-cov`, `ruff` are importable.
- `uv run ruff check src/` runs without error on the skeleton.

## Acceptance Criteria

- [ ] `uv sync --all-extras` exits 0.
- [ ] `uv run python -c "import crewai"` exits 0.
- [ ] `uv run ruff check src/` exits 0 on empty skeleton.
- [ ] `uv.lock` committed to version control.

## Notes / Constraints

- `pip install` is forbidden — all dependency operations must use `uv add` or `uv sync`.
- No `requirements.txt` file — `pyproject.toml` is the only dependency declaration.
- Project version in `pyproject.toml` must stay in sync with `shared/version.py` and all `config/*.json` version fields.
