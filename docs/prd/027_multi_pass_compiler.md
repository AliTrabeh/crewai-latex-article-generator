# Task 027 — Multi-Pass Compiler Orchestrator

## Goal

Implement `MultiPassCompiler` that runs the mandatory 4-pass compilation sequence (lualatex → biber → lualatex → lualatex) to fully resolve all cross-references and citations.

## Files to Create or Modify

- `src/latex_article_generator/services/compiler.py` — add `MultiPassCompiler` class

## Exact Expected Behavior

```python
from pathlib import Path

class MultiPassCompiler:
    def __init__(self, config) -> None:
        self._runner = LuaLatexRunner()
        self._biber = BiberRunner()

    def compile(self, latex_source: str, output_path: str) -> str:
        """
        Write latex_source to a temp .tex file, run 4-pass compilation,
        move PDF to output_path, return output_path.
        """
        import tempfile, shutil
        with tempfile.TemporaryDirectory() as tmp_dir:
            tex_file = str(Path(tmp_dir) / "article.tex")
            Path(tex_file).write_text(latex_source, encoding="utf-8")

            # Pass 1 — initial compile (generates .aux, .bcf)
            self._runner.run(tex_file, tmp_dir)
            # Pass 2 — biber processes bibliography
            self._biber.run(str(Path(tmp_dir) / "article"), tmp_dir)
            # Pass 3 — resolve bibliography references
            self._runner.run(tex_file, tmp_dir)
            # Pass 4 — resolve cross-references
            self._runner.run(tex_file, tmp_dir)

            pdf_src = str(Path(tmp_dir) / "article.pdf")
            Path(output_path).parent.mkdir(parents=True, exist_ok=True)
            shutil.copy2(pdf_src, output_path)

        return output_path
```

## Acceptance Criteria

- [ ] 4 passes are executed in order (verified by call count in tests).
- [ ] Output PDF is copied to `output_path`.
- [ ] If any pass fails, `CompilationError` propagates immediately (no further passes attempted).
- [ ] Temp directory is cleaned up even on failure (use `with tempfile.TemporaryDirectory()`).
- [ ] Unit tests mock `LuaLatexRunner.run` and `BiberRunner.run`.
- [ ] File ≤ 150 lines (shared with task 026 and 028).

## Notes / Constraints

- Exactly 4 passes — no more, no less. This is a hard requirement from the homework spec.
- Pass order: lualatex → biber → lualatex → lualatex.
- The `.bcf` file (generated by pass 1) is biber's input — biber reads it from the tmp directory.
- `shutil.copy2` preserves metadata and is safer than `shutil.move` across filesystem boundaries.
